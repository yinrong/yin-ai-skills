# 递归式深度调研 (recursive-research)

自动化深度研究任务，支持多轮迭代、来源分级、断点续传。适合技术选型、算法调研、文献综述等场景。

## 使用方法

```bash
/recursive-research <研究主题> [--resume <研究ID>]
```

## 工作流程

1. **初始化**：询问研究模式（网络/本地/混合）、输出目录、来源偏好
2. **拆解主题**：自动识别 3-5 条研究线索
3. **迭代研究**（文档驱动，先写后搜）：
   ```
   ⛔GATE1              ⛔GATE2
     ↓                    ↓
   [写]计划 → [做]搜集 → [写]发现+对比 → [写]对抗性审查 → 达标检查
     ↑                                                      ↓
     └──────────── 未达标：下一轮 ←──────────────────────────┘
   ```
   - **GATE1**：计划（为什么选这个方向+搜索策略+预期假设）未写完，不得开始搜索
   - **GATE2**：对抗性审查（偏差检查+来源独立性+方向修正）未完成，不得进入下一轮
   - **主动搜反面证据**：每轮至少 20% 精力寻找推翻当前结论的信息
4. **达标判定**：三维度同时满足
   - **广度**：线索覆盖度≥80%、每条线索≥3个独立T1来源、连续3轮饱和≤5%
   - **质量**：有反面证据记录、≥3跨线索连接、关键结论有交叉验证
   - **诚实性**：已声明适用范围和盲区、区分了强共识vs弱假设、知识缺口已填写

## 来源分级标准

| 等级 | 权重 | 标准 |
|------|------|------|
| **T1** | 5 | 学术论文、专著、标准文档（RFC/W3C/ISO）、官方数据集、归档资料 |
| **T2** | 3 | 官方代码仓库、专家博客（可引用）、会议录像、带引用的维基百科 |
| **T3** | 2 | 引用 T1/T2 的技术博客、有来源的高赞论坛回答、专家访谈 |
| **拒绝** | 0 | 无作者信息、无数据营销软文、SEO 垃圾、无监督 AI 生成内容 |

## 输出结构

研究结果保存在 `调研/<主题ID>/` 目录下：

```
调研/2026-06-02-分布式训练方案/
├── 状态.md              # 当前进度、下一步行动
├── 线索.md              # 3-5 条研究线索及覆盖度
├── 发现.md              # 按线索组织的关键发现
├── 来源-T1.md           # 一级来源清单
├── 来源-T2.md           # 二级来源清单
├── 来源-T3.md           # 三级来源清单
├── 来源-已拒绝.md        # 拒绝的来源及理由
├── 轮次-01.md           # 第 1 轮研究记录
├── 轮次-02.md           # 第 2 轮研究记录
├── ...
├── 轮次-N.md            # 第 N 轮研究记录
├── 综合报告.md          # 最终综合报告
├── 行动建议.md          # 可执行的后续行动
└── 知识缺口.md          # 未解决的问题
```

## 断点续传

研究被中断后，使用以下命令继续：

```bash
/recursive-research --resume 分布式训练方案
```

系统会从最后一个检查点加载状态继续研究。

## 决策框架

每个关键决策点使用 **加权决策矩阵 (WDM) + 芒格逆向思考**：

1. **WDM**：列举方案 → 设定加权标准 → 打分对比
2. **逆向思考**：
   - 如何让这个方案失败？
   - 我有什么认知偏差？
   - 我忽略了什么？

应用于：线索选择、来源筛选、结束判定、知识验证。

## 示例

```bash
# 新建调研
/recursive-research "Transformer 架构演进路径"

# 继续之前的调研
/recursive-research --resume transformer-架构演进路径
```

## 注意事项

- 每轮自动保存检查点，即使上下文压缩也能恢复
- 拒绝的来源会记录拒绝理由，避免重复评估
- 支持混合模式：优先使用本地文档，不足时联网补充
- 所有文件使用中文、Markdown 格式、UTF-8 编码
