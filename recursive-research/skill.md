# 递归式深度调研 (recursive-research)

自动化深度研究任务，支持多轮迭代、来源分级、断点续传。适合技术选型、算法调研、文献综述等场景。

## 使用方法

```bash
/recursive-research <研究主题> [--resume <研究ID>]
```

## 工作流程

1. **初始化**：询问研究模式（网络/本地/混合）、输出目录、来源偏好
2. **拆解主题**：自动识别 3-5 条研究线索
3. **迭代研究**：
   - 选择覆盖度最低的线索
   - 搜集并评估来源（分级为 T1/T2/T3/拒绝）
   - 整合发现到知识库
   - **保存检查点**
4. **达标判定**：满足以下所有条件后结束
   - 线索覆盖度 ≥80%
   - 每条线索至少 3 个 T1 级来源
   - 连续 3 轮新发现饱和度 ≤5%
   - 已应用逆向思考
   - 至少 3 个跨线索连接

## 来源分级标准

| 等级 | 权重 | 标准 |
|------|------|------|
| **T1** | 5 | 学术论文、专著、标准文档（RFC/W3C/ISO）、官方数据集、归档资料 |
| **T2** | 3 | 官方代码仓库、专家博客（可引用）、会议录像、带引用的维基百科 |
| **T3** | 2 | 引用 T1/T2 的技术博客、有来源的高赞论坛回答、专家访谈 |
| **拒绝** | 0 | 无作者信息、无数据营销软文、SEO 垃圾、无监督 AI 生成内容 |

## 输出结构

研究结果保存在 `调研/<主题ID>/` 目录下：

```

├── 状态.md              # 当前进度、下一步行动
├── 线索.md              # 3-5 条研究线索及覆盖度
├── 发现.md              # 按线索组织的关键发现
├── 来源-T1.md           # 一级来源清单
├── 来源-T2.md           # 二级来源清单
├── 来源-T3.md           # 三级来源清单
├── 来源-已拒绝.md        # 拒绝的来源及理由
├── 轮次-01.md           # 第 1 轮研究记录
├── 轮次-02.md           # 第 2 轮研究记录
├── ...

├── 行动建议.md          # 可执行的后续行动

```

## 断点续传

研究被中断后，使用以下命令继续：

```bash
/recursive-research --resume 分布式训练方案
```

系统会从最后一个检查点加载状态继续研究。

## 决策框架

每个关键决策点使用 **加权决策矩阵 (WDM) + 芒格逆向思考**：

1. **WDM**：列举方案 → 设定加权标准 → 打分对比
2. **逆向思考**：
   - 如何让这个方案失败？
   - 我有什么认知偏差？
   - 我忽略了什么？

应用于：线索选择、来源筛选、结束判定、知识验证。

## 示例

```bash
# 新建调研
/recursive-research "Transformer 架构演进路径"

# 继续之前的调研
/recursive-research --resume transformer-架构演进路径
```

## 注意事项

- 每轮自动保存检查点，即使上下文压缩也能恢复
- 拒绝的来源会记录拒绝理由，避免重复评估
- 支持混合模式：优先使用本地文档，不足时联网补充
- 所有文件使用中文、Markdown 格式、UTF-8 编码
